# Project Instructions

## 项目性质

这是飞书/多维表格仪表盘插件项目，不是普通 Web 项目。任何功能开发、依赖升级、构建配置调整、SDK 调用改造和 UI 交互改动，都必须先按多维表格仪表盘插件规范判断兼容性。

## 必读资料

开发前先参考本项目内的插件开发资料：

- `docs/多维表格仪表盘插件开发资料/README.md`
- `docs/多维表格仪表盘插件开发资料/docs/多维表格-仪表盘插件-开发指南.md`
- `docs/多维表格仪表盘插件开发资料/docs/Base-JSSDK-Dashboard-插件-API-文档.md`
- `docs/多维表格仪表盘插件开发资料/docs/Base-开放设计规范.md`
- `docs/多维表格仪表盘插件开发资料/sdk/`
- `docs/多维表格仪表盘插件开发资料/examples/`

如果实现涉及 SDK API、dashboard 环境、插件发布、移动端兼容、主题样式或 Base 数据读写，必须先查上述资料，再改代码。

## 开发约束

- 不要把本项目当成独立普通网页应用来设计运行时能力；必须考虑多维表格仪表盘插件容器、SDK 能力边界和发布要求。
- 不要擅自替换 `@lark-base-open/js-sdk`、Vite 插件配置或插件运行入口；确需调整时先确认资料中的兼容规范。
- 不要通过兜底逻辑掩盖 SDK、权限、字段类型、记录读写或插件容器兼容问题；优先定位根因并让错误暴露出来再修正。
- UI 改动需参考 Base 开放设计规范和 Semi/飞书仪表盘主题，避免做成普通营销页或通用后台页。
- 示例数据和本地 mock 只能用于开发验证；涉及真实 Base 表字段、记录、写回或缓存策略时，必须以插件 SDK 和实际表结构为准。

## 验证建议

常规改动后至少运行：

```bash
npm run build
```

涉及核心逻辑、数据解析、写回或缓存时同时运行相关测试：

```bash
npm test
```
